## Summary

Deliver the Tenantra admin operations dashboard as the authoritative control center for property managers. The
feature spans automated monthly billing, unit and resident lifecycle management, resilient announcements, facility
booking, analytics, and immutable audit logging in a Turborepo-managed monorepo. Implementation emphasizes audit-able
data flows, versioned shared packages (`packages/api`, `packages/db`, `packages/ui`), and admin-only access backed by
NextAuth.js and Prisma.

### Implementation Pillars

- **Admin-First Delivery**: Prioritize administrator billing, communication, and analytics workflows with staging data
  parity before opening mobile integrations.
- **Shared Domain Packages**: Centralize business logic in `packages/api` and `packages/db` so future apps consume
  consistent contracts.
- **Audit & Observability**: Every critical action (billing, announcements, bookings) writes to `AuditLogEntry` and is
  captured by structured logging forwarders.

seeding utilities  
**Target Platform**: Vercel-hosted web admin (apps/web) with API routes deployed alongside; future Expo mobile app
consuming shared API  
**Project Type**: Monorepo (`apps/` + shared `packages/`) managed by Turborepo  
**Performance Goals**: KPI dashboard ≤2s p95 load for 5k units; invoice batch generation completes ≤5 minutes; email
delivery status updates within 60 seconds; audit writes within 10 seconds  
**Constraints**: Enforce RBAC (admin only), immutable audit logs, automated billing reconciliation, communication retry
policies, observability instrumentation before feature enablement  
**Scale/Scope**: Target 5k units / 10k residents initial rollout with growth to 20k units; concurrent admin sessions <50
with burst handling for billing operations
